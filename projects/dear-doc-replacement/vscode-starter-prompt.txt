I am building a WordPress plugin MVP called "Prism Chat Widget" to replace DearDoc with a scripted patient intake flow.

IMPORTANT CONTEXT
- This is an MVP / prototype-first build.
- Use TEST DATA ONLY during development.
- Do not claim HIPAA compliance in code comments or docs.
- Build with security-conscious defaults, but treat production hardening as a later step.
- This is NOT a freeform AI chatbot.
- This is a scripted intake widget.
- No EHR integration.
- No appointment booking.
- Bitrix24 integration is required, but phase 1 can stub it behind a service class.

PRODUCT GOAL
Create a WordPress plugin that renders a chat-style intake widget on the site and collects:
- location
- new vs existing patient
- full name
- phone number
- SMS consent
- email
- reason for visit
- insurance provider
- preferred appointment date/time

TECHNICAL STACK
- WordPress plugin
- PHP 8+
- vanilla JavaScript
- CSS
- WordPress Settings API for admin settings
- WordPress REST API for form submission
- modular class-based structure where reasonable
- no React, no build step, no external JS frameworks

PLUGIN NAME
Prism Chat Widget

PLUGIN SLUG
prism-chat-widget
