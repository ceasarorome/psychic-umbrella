# DearDoc Replacement Project Package

This folder now holds the core planning package for replacing DearDoc with a self-hosted, scripted intake/chat workflow.

## What is included

- `mvp-brief.md` - product direction and MVP recommendation
- `../../brief-deardoc-replacement.md` - broader compliance and delivery brief
- `../../job-description-chat-developer.md` - job post for hiring a developer
- `../../workflow-build-chat-widget.txt` - do-it-yourself build workflow for VS Code
- `developer-evaluation-scorecard.md` - rubric for comparing proposals and screening candidates

## Recommended order

1. Read `mvp-brief.md` to choose the product shape
2. Read `../../brief-deardoc-replacement.md` to confirm compliance constraints
3. Decide whether this is:
   - a DIY prototype, or
   - a production build that should be handed to a developer
4. If hiring, post `../../job-description-chat-developer.md`
5. Score responses with `developer-evaluation-scorecard.md`
6. If building yourself first, follow `../../workflow-build-chat-widget.txt`

## My recommendation

The best path is probably:

1. use the DIY workflow to clarify the flow, fields, and Bitrix24 mapping
2. build or fake a simple prototype if you want hands-on confidence
3. hire a developer for the production-grade version, especially the security and deployment pieces

That keeps costs down without winging the compliance-sensitive parts.
